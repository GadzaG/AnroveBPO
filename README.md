# AnroveBPO

Проект Web API, собранный по слоистой архитектуре с разделением ответственности между доменной моделью, бизнес-логикой, инфраструктурой и веб-слоем.

## Что успел

За текущий этап сделано следующее:

- оформлены основные слои решения;
- реализована работа с доменными сущностями и value object;
- настроена работа с PostgreSQL и миграциями;
- добавлена аутентификация через JWT и роли на базе `AspNetCore.Identity`;
- подключен Swagger;
- добавлен стартовый сидинг администратора;
- реализован единый формат ответов API через `EndpointResult`.

## Архитектура по слоям

### 1. `AnroveBPO.Domain`

Содержит доменную модель:

- корневые сущности;
- value object;
- собственную модель ошибок (`Error`, `GeneralErrors`);
- пользовательские exceptions.

### 2. `AnroveBPO.Application`

Слой бизнес-логики:

- use-case обработчики;
- интерфейсы репозиториев;
- `ITransactionManager` (Unit of Work) и `ITransactionScope` (подготовлен, но в текущем коде не использован);
- CQS-подход: разделение обработчиков чтения и записи;
- автоматическая регистрация обработчиков в DI через `Scrutor`.

### 3. `AnroveBPO.Infrastructure.Postgres`

Инфраструктура для основной БД:

- реализации репозиториев;
- `DbContext`;
- конфигурации таблиц;
- миграции;
- DI-расширение для подключения слоя.

### 4. `AnroveBPO.Infrastructure.Identity`

Инфраструктура для пользователей и авторизации:

- `AspNetCore.Identity`;
- генерация JWT-токенов;
- роли пользователей;
- DI-расширение.

Примечание: связка `Identity User` и доменной модели `Customer` задумана, но до конца не доведена в рамках текущего этапа.

### 5. `AnroveBPO.Web`

Слой представления (Web API):

- контроллеры и endpoint'ы;
- конфигурация приложения и DI;
- middleware для обработки необработанных исключений;
- Swagger/OpenAPI;
- собственный `EndpointResult` (реализация поверх `IResult`) для единообразного формата ответов.

### 6. Все модели представления выносил в Anrove.BPO.Contracts, который не ссылается ни на один проект.

## Авторизация и роли

- `email/username`: `admin`
- `password`: `admin`
- роль: `admin`

## Запуск проекта

```bash
docker compose up --build
```

После запуска:

- API: `http://localhost:3001`
- Swagger: `http://localhost:3001/swagger/index.html`
- PostgreSQL: `localhost:3000`
